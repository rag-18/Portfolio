CipherHub

Cybersecurity Community Forum · Feb – Mar 2026 · ~2,500 lines Python
// THE PROBLEM

Most portfolio apps treat security as an add-on. I wanted to build a system where security was the primary design principle at every layer of the stack — then actually attack it to prove it holds.

// WHAT I BUILT

A fully-featured cybersecurity community forum (posts, rich-text editor, comments, DMs, notifications, follow system, reputation ranks, full-text search) with 17 distinct security controls covering 9/10 OWASP Top 10 2021 categories, a custom real-time IDS, and a staff-only SOC Security Dashboard.

// KEY SECURITY DECISIONS
  • Custom IDS middleware scans every HTTP request for SQLi, XSS, path traversal, command injection — returns 403 and logs to AttackLog model
  • TOTP 2FA via Google Authenticator — stolen passwords alone cannot compromise accounts
  • Rate limiting (5 req/min) + lockout after 5 failures — stops brute force and credential stuffing
  • PBKDF2-SHA256 at 720k iterations — GPU cracking practically infeasible
  • CSP headers + bleach sanitization on all user input to neutralize XSS
// TESTING & VALIDATION
  • 8 manual pen test scenarios — all blocked · OWASP ZAP: 16 alerts, all documented
  • pip-audit: 7 CVEs found and remediated — final result: 0 known vulnerabilities
  • STRIDE: 28 threats across 5 components, all High/Medium mitigated
